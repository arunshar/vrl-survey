\section{A Formal Framework for Verbal Reinforcement Learning}
\label{sec:framework}

The surveyed methods share a decision-theoretic skeleton but attach language to different components. We define that skeleton, identify where a linguistic artifact can enter it, and state the two criteria that separate verbal reinforcement learning (VRL) from neighboring paradigms.

\subsection{The Language-Augmented Decision Process}
\label{subsec:setting}

Begin with a POMDP $\mathcal{M}=(\mathcal{S},\mathcal{A},\Omega,T,R,O,\gamma)$ \citep{kaelbling1998planning,sutton2018reinforcement}. Let $\mathcal{L}$ be the set of finite natural-language strings. A \emph{linguistic artifact} $\ell\in\mathcal{L}$ is a string produced or consumed as a signal, such as a task description, critique, rationale, stored lesson, or reward-code fragment. The agent configuration $\kappa=(\theta,c,m)$ comprises model parameters $\theta$, working context $c$, and persistent memory $m$; policy $\pi_\kappa$ maps interaction histories to action distributions.

The construct that distinguishes VRL from ordinary language-conditioned control is an \emph{improvement operator}
\begin{equation}
\mathcal{I}:(\kappa,\ell)\longmapsto\kappa',
\label{eq:improvement}
\end{equation}
which consumes an artifact and modifies the agent. Three realizations recur. An \emph{output revision} changes $c$, for example by appending critique and regenerating \citep{madaan2023selfrefine}. A \emph{memory write} changes $m$, preserving a lesson across episodes without changing weights \citep{shinn2023reflexion,zhou2025memento}. A \emph{parameter update} changes $\theta$ and therefore future behavior. Operators may compose these effects, as when critique-generated revisions later become training data \citep{bai2022constitutional}.

Language can enter the augmented process at six points.
\begin{enumerate}[label=(\roman*),leftmargin=2.2em,itemsep=0.2em]
    \item \emph{Goal specification.} A task description $g\in\mathcal{L}$ defines success when the true objective is latent.
    \item \emph{State and observation.} Text environments, tool outputs, error traces, or verbalized sensor state provide the observation substrate.
    \item \emph{Action prior.} Instructions or generated subgoals reshape the action distribution before learning \citep{he2024words}.
    \item \emph{Reward.} A generator may compile $\ell$ into $\hat R=C(\ell)$ at specification time \citep{ma2024eureka}, or a judge may evaluate trajectories at runtime \citep{zheng2023judging}.
    \item \emph{Transition prior.} Pretrained knowledge supplies predictions about action consequences, elicited in language \citep{he2024words}.
    \item \emph{Improvement.} Critiques, reflections, and rationales serve directly as the argument to $\mathcal{I}$.
\end{enumerate}

The taxonomy maps signals acting on points (i) through (iv) at problem-definition time to grounding; output revisions and memory writes at inference time to deliberative feedback; and parameter updates to learning signals. The transition prior is a formal coordinate that the surveyed literature does not yet occupy with an explicit artifact. This POMDP is an organizing vocabulary, not a claim that every agent setting supplies a known transition kernel, reward function, Markov state, or stationary environment. In open-ended tool use and dialogue, those objects may be latent or undefined, so we do not import POMDP theorems merely because the notation is useful.

\subsection{The Definitional Fence}
\label{subsec:fence}

A method belongs to VRL if and only if two conditions hold. First, feedback is expressed in natural language and its linguistic structure is functionally consumed rather than collapsed to a scalar or preference bit before use. Second, the feedback participates in an improvement loop through output revision, memory write, parameter update, or construction of a decision-problem component that later drives one of those updates.

Criterion (i) can be tested with a scalarization map $\sigma:\mathcal{L}\to\mathbb{R}^{k}$. If replacing $\ell$ by $\sigma(\ell)$ everywhere leaves the distribution over updated configurations $\kappa'$ unchanged, the method does not functionally consume the language. Here $k$ is fixed per judged unit: a preference bit, grade, rubric vector, or score per reasoning step is scalarized even when a trajectory contains many steps. Retaining text is not. Criterion (ii) requires that $\ell$ enter $\mathcal{I}$ directly or construct a component of $\mathcal{M}$ that subsequently drives improvement.

Because this test is counterfactual, we apply a dataflow inspection rule. Trace the artifact from producer to consumer and mark its \emph{scalarization boundary}, where linguistic structure collapses into the numbers received by the learner. The criterion holds if a downstream consumer reads the artifact's structure, such as a compiler parsing reward code \citep{ma2024eureka}, a reviser conditioning on critique \citep{madaan2023selfrefine}, a retriever matching stored lessons \citep{shinn2023reflexion}, or a loss computed over feedback text. It fails when the sole downstream interface receives only a score. A rationale-annotated preference therefore qualifies only if the rationale enters revision or training; a step critique qualifies only if its text, rather than only its score, reaches the improvement machinery. The rule settles mechanism membership while leaving an empirical question: whether a consumer capable of reading linguistic structure actually exploits it.

Both criteria exclude familiar neighbors. Plain instruction following fails criterion (ii), since it conditions behavior without an improvement loop. Vanilla RLHF \citep{ouyang2022training} and DPO \citep{rafailov2023direct} fail criterion (i) when the annotator's rationale is discarded before the update. Language-conditioned RL \citep{luketina2019survey} uses language as policy input while scalar reward drives improvement. Reward-code generation qualifies because a compiler consumes the full artifact before the resulting function trains a policy \citep{ma2024eureka}. The feedback must be text, but observations need not be, which admits embodied agents receiving verbal corrections.

\subsection{What Is Formally Guaranteed, and What Is Not}
\label{subsec:guarantees}

Formal results are recent and localized. \citet{xu2025formalizing} formalize learning from language feedback under latent rewards, define the transfer eluder dimension as a measure of feedback informativeness, and give the no-regret HELiX algorithm. Their separation result shows that rich language feedback can be exponentially more sample efficient than reward alone under their assumptions. \citet{he2024words} analyze hierarchical planning inside a POMDP: imitation of language subgoals can incur linear regret, while exploration restores sublinear regret when pretraining error is small. LLF-Bench operationalizes a related setting by replacing numeric rewards with varied natural-language instructions and feedback \citep{cheng2023llfbench}.

These results do not establish a general theory of VRL. No policy-improvement theorem covers $\mathcal{I}$, no convergence result covers language-space Bellman operators, and no bound covers self-generated, noisy, strategic, or adversarial feedback. Existing separations compare rich language with scalar reward but do not characterize intermediate formats such as step critiques or rationale-annotated preferences. Criterion (i) also lacks a measurable certificate of how much linguistic structure a learner uses. The framework therefore makes its claim boundary explicit: it supports classification and exposes the missing quantities, but it does not guarantee that critique, memory, or language-shaped reward improves a policy.
